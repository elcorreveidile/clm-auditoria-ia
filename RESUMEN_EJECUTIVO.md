# 🎯 RESUMEN EJECUTIVO
## Reestructuración del Formulario de Auditoría IA

---

## 📊 ANTES vs DESPUÉS

### ESTRUCTURA ACTUAL (3 caminos)
```
┌─────────────────────────────────────────┐
│ docente   → Profesores (ELE + LM)      │
│ admin     → Admin + Marketing + Conserj │
│ tecnico   → Diseño + Informática + Dir │
└─────────────────────────────────────────┘
❌ Marketing comparte con Administración
❌ Dirección comparte con Informática
❌ Informática agrupa 3 roles diferentes
```

### NUEVA ESTRUCTURA (7 caminos)
```
┌────────────────────────────────────────────────────┐
│ 1. docente     → Profesores (ELE + LM)            │
│ 2. admin       → Administración y Gestión          │
│ 3. marketing   → Marketing y Comunicación    ✨    │
│ 4. conserjeria → Conserjería y Mantenimiento  ✨    │
│ 5. disenyo     → Diseño Gráfico y Audiovisual  ✨  │
│ 6. desarrollo  → Programación y Desarrollo    ✨   │
│ 7. sistemas    → Sistemas y Seguridad Informática │
│ 8. direccion   → Dirección y Subdirecciones   ✨   │
└────────────────────────────────────────────────────┘
✅ Cada área tiene su propio camino específico
✅ Preguntas adaptadas a cada rol
✅ Informática separada en 3 especialidades
```

---

## 🆕 NUEVOS CAMINOS CREADOS

### 3️⃣ MARKETING (4 bloques)
```
M1: Uso de IA en Marketing
   ├── Plataformas que gestionas
   ├── Frecuencia de uso
   ├── Nivel de prompting
   └── Privacidad de datos

M2: Contenido y Redes Sociales
   ├── Tareas automatizables (copy, imágenes, vídeos)
   ├── Análisis de métricas
   ├── Programación de contenido
   └── Agentes para redes sociales

M3: Actitudes y Formación
   ├── Percepción de la IA en marketing
   ├── Preocupaciones éticas
   └── Necesidades de formación específica
```

### 4️⃣ CONSERJERÍA Y MANTENIMIENTO (4 bloques)
```
C1: Uso real de IA
   ├── Tareas que realizas (mantenimiento, atención)
   ├── Nivel digital
   └── Uso actual de IA

C2: Automatización y Gestión
   ├── Tareas administrativas repetitivas
   ├── Partes de mantenimiento digitalizados
   └── Asistente por voz/foto para incidencias

C3: Actitudes y Formación
   ├── Percepción de IA en su trabajo
   ├── Preocupaciones sobre automatización
   └── Formación digital básica
```

### 5️⃣ DISEÑO GRÁFICO Y AUDIOVISUAL (4 bloques)
```
DG1: Uso de IA en Diseño
   ├── Herramientas creativas (Midjourney, Firefly, Runway)
   ├── Integración en flujo de trabajo
   └── Nivel de adopción

DG2: Proyectos Creativos
   ├── Tareas automatizables (imágenes, vídeo, audio)
   ├── Porcentaje de trabajo asistible
   └── Frenos y barreras

DG3: Actitudes y Formación
   ├── Impacto en la profesión
   ├── Preocupaciones sobre originalidad
   └── Formación en IA generativa
```

### 6️⃣ PROGRAMACIÓN Y DESARROLLO (4 bloques)
```
DEV1: Uso de IA en Desarrollo
   ├── Code assistants (Copilot, Claude Code)
   ├── Integración de APIs
   └── Casos de uso avanzados

DEV2: Proyectos e Infraestructura
   ├── Tipo de proyectos en el CLM
   ├── Infraestructura de desarrollo
   └── Riesgos específicos de código

DEV3: Visión Estratégica
   ├── Integración de IA en proyectos
   ├── Seguridad en código generado
   └── Liderazgo técnico
```

### 7️⃣ SISTEMAS Y SEGURIDAD INFORMÁTICA (4 bloques)
```
SYS1: Uso de IA en Sistemas
   ├── Responsabilidades (ENS, seguridad)
   ├── Uso actual para diagnóstico
   └── Nivel de conocimiento

SYS2: Infraestructura y Seguridad
   ├── Infraestructura existente
   ├── Cumplimiento del ENS
   ├── Riesgos de seguridad específicos
   └── Sistemas de atención al estudiante

SYS3: Visión Estratégica
   ├── Preparación del CLM para IA
   ├── Implantación progresiva
   └── Liderazgo técnico
```

### 8️⃣ DIRECCIÓN Y SUBDIRECCIONES (4 bloques)
```
DIR1: Visión Estratégica
   ├── Responsabilidades directivas
   ├── Nivel de conocimiento de IA
   └── Oportunidades identificadas

DIR2: Gestión y Toma de Decisiones
   ├── Áreas de decisión con IA
   ├── Datos disponibles para análisis
   ├── Agentes recomendadores
   └── Barreras para datos

DIR3: Política Institucional
   ├── Prioridad estratégica
   ├── Riesgos institucionales
   ├── Rol de la dirección
   └── Asignación de presupuesto
```

## 🎨 SEPARACIÓN DE INFORMÁTICA (3 roles)

### ANTES: Todo junto en `tecnico`
```
tecnico → Diseño + Informática + Programación + Dirección
          ❌ Preguntas técnicas para directivos
          ❌ Preguntas de estrategia para programadores
```

### DESPUÉS: 3 caminos especializados
```
disenyo    → Diseño Gráfico y Audiovisual
desarrollo → Programación y Desarrollo
sistemas   → Sistemas y Seguridad Informática
           ✅ Cada rol con preguntas específicas
```

## 📈 IMPACTO DE LA MEJORA

### Calidad de Datos
| Aspecto | Antes | Después |
|---------|-------|---------|
| Perfiles | 3 genéricos | 7 específicos |
| Preguntas relevantes | ~60% | ~95% |

| Aspecto | Antes | Después |
|---------|-------|---------|
| "No aplica" | Frecuente | Raro |
| Tiempo de respuesta | ~15 min | ~12 min |
| Satisfacción esperada | Media | Alta |

### Valor para la Organización
| Aspecto | Antes | Después |
|---------|-------|---------|

| Identificación de necesidades | General | Precisa |
| Plan de formación | Genérico | Personalizado |
